# Практика 201: VSM и инициализация DevOps-проекта

## Название проекта
**DevOps Learning API** — простой REST API для демонстрации DevOps практик

## Описание
Веб-приложение на Flask, которое предоставляет базовые эндпоинты для работы с пользователями. Используется как основа для изучения CI/CD, контейнеризации и мониторинга.

## Технологический стек
- **Язык:** Python 3.14
- **Фреймворк:** Flask 3.0.0
- **База данных:** SQLite (планируется PostgreSQL в П203)
- **Контейнеризация:** Docker (планируется в П203)
- **CI/CD:** GitHub Actions (планируется в П202)
- **Облако:** AWS или DigitalOcean (выбор в П208)
- **ОС:** Windows 11, но кроссплатформенное

## Результаты аудита потока (AS-IS)

### Lead Time
- **Текущий Lead Time:** ~3-4 часа (от идеи до deploy)
- **Разбивка:**
  - Разработка: 30 мин
  - Тестирование: 30 мин
  - Ревью кода: 1-2 часа (ожидание)
  - Деплой: 30 мин

### Главные «боли» процесса
1. **Ожидание код-ревью (самое долгое)** — нет автоматизации, все ручное
2. **Ручной деплой на сервер** — нет CI/CD пайплайна
3. **Отсутствие unit-тестов** — нечего автоматизировать
4. **Нет мониторинга** — неизвестно, когда приложение падает
